Ce qui donne genre :
Code:
unzip prupraprok-1.zip -d prupraprok
cd prupraprok
patch -p1 < ~/prupraprok.patch
cd code_source/splash
make
cd ../../prupraprok
cp ../../code_source/splash/splash .
./splash film1.zob


Par contre, le parseur XML est bogué : pb retour à la ligne Windows/Linux !? J'ai viré tous les retours à la ligne dans les fichiers .zob et ça passe :
Code:
tr -d $(echo -ne "\r") < film1.zob | tr -d $(echo -ne "\r") > film1-linux.zob
tr -d $(echo -ne "\r") < film2.zob | tr -d $(echo -ne "\r") > film2-linux.zob


Pour la compilation, il faut les bibliothèques + les entêtes de développement pour SDL, SDL_image, SDL_mixer et zlib.

C'est laborieux, mais ça permet de voir le film.

Code:
